## 5. 数据一致性检查

### 5.1 核心实体对比

| 实体 | docs/03 PRD §11 | docs/07 §4.5 | docs/08 DB §2 | 冲突？ |
|------|----------------|-------------|---------------|--------|
| User（学生） | User (Id, Name, Role, Password) | User (Id, StudentID, DisplayName) | Users (Id, StudentId, DisplayName) | 🟡 见下文 |
| Admin（管理员） | 合并在 User.Role | Admin (Id, Username, PasswordHash) | Admins (Id, Username, PasswordHash) | 🟡 见下文 |
| SeatArea | SeatArea (Id, Name, Description) | SeatArea | SeatAreas (Id, Name, Floor, SortOrder) | ✅ |
| Seat | Seat (Id, SeatNumber, AreaId, Floor, HasPower, HasLight, Status) | Seat | Seats (Id, SeatNumber, AreaId, Status, HasPower, HasLight) | ✅ |
| Reservation | Reservation (Id, UserId, SeatId, TimeSlotId, Date, Status, CreatedAt) | Reservation | Reservations (Id, UserId, SeatId, Date, TimeSlot VARCHAR, Status, CreatedAt, CancelledAt) | 🔴 见下文 |
| **TimeSlot** | **独立实体** (Id, Name, StartTime, EndTime) | 未显式列出 | **不建独立表** → VARCHAR 字段 | 🔴 见下文 |

### 5.2 关键分歧点

#### 🔴 分歧 1：TimeSlot 实体存在性

- **docs/03 PRD §11**：TimeSlot 是独立实体，含 Id/Name/StartTime/EndTime，与 Reservation 是 1:N 关系
- **docs/08 DB §2**：时段为预定义固定 7 个值，不建独立表，以 VARCHAR `"08:00-10:00"` 存于 Reservations 表
- **docs/07 §4.5**：未显式列出 TimeSlot 实体（隐含简化）

**判定**：下游（docs/07 + docs/08 + 任务板 T00-04）已统一采用"无独立 TimeSlot 表"方案。docs/03 需要回改 §11 核心数据对象，将 TimeSlot 从独立实体改为"预定义固定值说明"。

#### 🟡 分歧 2：Admin 实体分离

- **docs/03 PRD §11**：管理员合并到 User 实体，通过 `Role=Admin` 区分，共用一张 Users 表
- **docs/07 §4.5**：Admin 独立为一个实体（轻量登录，非 Identity）
- **docs/08 DB §2**：Admins 独立建表，字段为 Id/Username/PasswordHash/CreatedAt

**判定**：下游（docs/07 + docs/08）采用了 Admin 独立建表方案（更轻量、职责清晰）。docs/03 需要回改 §11 和 §12 中的用户模型描述。

#### 🟡 分歧 3：Reservation 字段差异

- **docs/03 PRD**：Reservation 含 `TimeSlotId`（FK 指向 TimeSlot 表）
- **docs/08 DB**：Reservation 含 `TimeSlot`（VARCHAR，如 `"08:00-10:00"`）+ `CancelledAt`（datetime）

**判定**：这是分歧 1 的直接后果。docs/03 需同步更新 Reservation 字段定义。
