# 图书馆座位预约系统 — PRD Lite

## 1. 项目概述

**图书馆座位预约系统**是一个轻量级的在线座位预约工具，面向在校学生和图书馆管理员。学生可以通过手机或电脑浏览图书馆座位、提交预约、查看和取消自己的预约；管理员可以在后台管理座位信息、查看全部预约记录、查看简单的统计数据。系统采用 ASP.NET Core MVC (.NET 8) 架构，数据库使用 SQL Server LocalDB + EF Core，前端使用 Bootstrap 响应式布局。

## 2. 项目目标

1. 学生能完成"浏览座位 → 查看详情 → 选择日期时段 → 提交预约 → 查看我的预约 → 取消预约"的完整闭环。
2. 管理员能完成"登录 → 查看全部预约并筛选 → 管理座位（增/改/停用）→ 查看今日统计"的管理闭环。
3. 同一座位同一日期同一时段不允许重复预约（核心业务规则）。
4. 系统代码结构清晰、数据库设计合理，可作为课程作业展示。

## 3. 目标用户

| 用户类型 | 角色 | 核心诉求 |
|----------|------|----------|
| 学生用户 | 预约者 | 快速找到空位、提交预约、查看已约座位、临时取消 |
| 管理员 | 系统管理者 | 管理座位状态、查看预约全貌、掌握当日使用情况 |

## 4. 使用场景

| 编号 | 场景 | 用户动作 |
|------|------|----------|
| SC-01 | 考前预约 | 学生打开系统 → 浏览座位列表 → 点击自习区座位查看详情 → 选择"明天上午" → 提交预约 → 成功 |
| SC-02 | 课间临时预约 | 学生切换到体验账号 → 看座位列表 → 发现有空位 → 预约"今天下午" |
| SC-03 | 临时取消 | 学生打开"我的预约" → 找到明天上午的预约 → 点击"取消" → 确认 → 取消成功 |
| SC-04 | 管理员巡检 | 管理员登录 → 进入预约管理 → 筛选今天日期 → 查看全部预约 → 进入统计页看分布 |
| SC-05 | 座位维护 | 管理员登录 → 座位管理 → 找到故障座位 → 点击"停用" → 该座位在用户端变为不可约 |

## 5. 用户端核心功能

| 功能 | 说明 | 最小验收标准 |
|------|------|-------------|
| **体验账号切换** | 页面顶部下拉框，预设 3 个学生账号，切换即生效 | 切换后，我的预约列表变更为该账号的数据 |
| **用户首页** | 展示图书馆名称、开放时间、快捷按钮"找座位"和"我的预约" | 页面加载后两个按钮可点击跳转 |
| **座位列表页** | 按区域分组展示座位，每张座位卡片显示座位号、区域、状态（可预约/已满/停用） | 至少展示 2 个区域、每个区域至少 3 个座位；状态与数据库一致 |
| **座位详情页** | 展示座位号、区域、楼层、设施（电源/台灯）、当前日期可预约时段 | 从座位列表点进来后信息完整；有"去预约"按钮 |
| **预约提交页** | 显示已选座位，用户选择日期和时段，点击提交 | 提交成功后跳转"我的预约"并看到新记录 |
| **预约冲突检查** | 提交时后端检查：同一座位+同一日期+同一时段是否已有有效预约 | 重复预约时返回错误提示，不生成记录 |
| **我的预约页** | 按时间倒序列出当前用户的全部预约，每条显示座位号、日期、时段、状态 | 状态标签颜色区分（绿色=已预约、灰色=已取消） |
| **取消预约** | 在"我的预约"中点击"取消"，弹出确认，确认后状态变为"已取消" | 取消后该时段释放，其他用户可以预约 |

## 6. 管理端核心功能

| 功能 | 说明 | 最小验收标准 |
|------|------|-------------|
| **管理员登录** | 输入账号密码，验证通过后进入管理后台 | 错误密码提示"账号或密码错误"；正确密码跳转预约管理页 |
| **预约管理** | 表格展示全部预约记录，支持按日期、状态、区域筛选 | 筛选条件切换后表格即时更新；列表显示用户、座位、日期、时段、状态 |
| **座位管理** | 表格展示全部座位，支持新增座位、编辑座位信息、停用/启用 | 新增后列表即时出现；停用后用户端不再显示为可约 |
| **统计页** | 顶部数据卡片展示今日预约总数；下方表格展示各区域预约数 | 今日预约总数与预约管理页筛选结果一致 |

## 8. 核心业务流程

### 8.1 用户预约主流程

```
[用户首页] → 点击"找座位" → [座位列表页] → 点击某个座位
→ [座位详情页] → 点击"去预约" → [预约提交页] → 选日期+时段 → 提交
→ 后端检查冲突 → 写入数据库 → 跳转 [我的预约页]（显示新记录）
```

### 8.2 用户取消流程

```
[我的预约页] → 找到目标预约 → 点击"取消" → 弹出确认 → 确认
→ 后端更新状态为"已取消" → 刷新页面
```

### 8.3 管理端主流程

```
[管理员登录页] → 输入账号密码 → 验证通过 → [预约管理页]
→ 按日期/状态筛选 → 查看预约列表
→ 导航到 [座位管理页] → 新增/编辑/停用座位
→ 导航到 [统计页] → 查看今日预约数据
```

## 10. 验收标准

| 编号 | 验收项 | 验收方法 |
|------|--------|----------|
| AC-01 | 用户主链路闭环 | 从首页→座位列表→详情→提交预约→我的预约，全流程可走通 |
| AC-02 | 预约冲突拦截 | 对已预约的座位+日期+时段再次提交，系统拒绝并提示 |
| AC-03 | 取消后释放 | 取消预约后，该座位+日期+时段可被其他账号预约 |
| AC-04 | 体验账号切换生效 | 切换账号后，我的预约数据随之切换 |
| AC-05 | 管理员登录控制 | 未登录直接访问后台页面时重定向到登录页 |
| AC-06 | 预约管理筛选 | 按日期筛选后列表只显示当日预约；按状态筛选同理 |
| AC-07 | 座位停用生效 | 管理员停用某座位后，用户端座位列表该座位显示"停用"且不可预约 |
| AC-08 | 统计数据准确 | 统计页今日预约总数 = 预约管理页筛选今日后的记录数 |

## 11. 核心数据对象

| 实体 | 核心字段 | 说明 |
|------|----------|------|
| **User** | Id, Name, Role(Student/Admin), Password | 用户（学生体验账号 + 管理员） |
| **SeatArea** | Id, Name, Description | 座位区域（如"自习区A"、"电子阅览区"） |
| **Seat** | Id, SeatNumber, AreaId, Floor, HasPower, HasLight, Status(Available/Disabled) | 座位信息 |
| **TimeSlot** | Id, Name, StartTime, EndTime | 时段定义（如 08:00-12:00、13:00-17:00、18:00-22:00） |
| **Reservation** | Id, UserId, SeatId, TimeSlotId, Date, Status(Active/Cancelled), CreatedAt | 预约记录 |

**关系**：User 1:N Reservation；SeatArea 1:N Seat；Seat 1:N Reservation；TimeSlot 1:N Reservation。

## 12. 状态流转说明

### 12.1 预约状态流转

```
[无预约] → 用户提交预约 → [已预约(Active)]
[已预约(Active)] → 用户取消 → [已取消(Cancelled)]
[已预约(Active)] → 管理员取消 → [已取消(Cancelled)]
```

注意：已取消的预约不可恢复，只能重新预约。

### 12.2 座位状态流转

```
[可用(Available)] → 管理员停用 → [停用(Disabled)]
[停用(Disabled)] → 管理员启用 → [可用(Available)]
```

停用状态下的座位不可预约。已有预约不受座位停用影响（已约的仍然有效）。

## 15. 风险与假设

| 风险 | 应对 |
|------|------|
| 时间不足 | 严格锁定 9 页，不扩展功能 |
| Bootstrap 默认样式不够"像样" | 补一份自定义 CSS，但不过度追求视觉 |
| EF Core Migration 出错 | 前期用简单一对一/一对多关系，避免多对多 |
| 预约并发冲突 | 本期用数据库唯一约束 + 后端检查，不做分布式锁 |
| AI 代码不稳定 | 每个 Sprint 完成后对照验收标准手动测试 |

**假设**：开发环境 Windows 10/11，已安装 .NET 8 SDK、SQL Server LocalDB、Visual Studio 2022 或 VS Code。
