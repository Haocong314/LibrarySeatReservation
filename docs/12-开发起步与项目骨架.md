### 1.2 工程创建命令序列

```powershell
# 1. 创建解决方案和 Web 项目
dotnet new sln -n LibrarySeatReservation --output src
cd src
dotnet new mvc -n LibrarySeatReservation.Web --framework net8.0
dotnet sln add LibrarySeatReservation.Web/LibrarySeatReservation.Web.csproj

# 2. 安装 NuGet 包
dotnet add LibrarySeatReservation.Web package Microsoft.EntityFrameworkCore.SqlServer
dotnet add LibrarySeatReservation.Web package Microsoft.EntityFrameworkCore.Design
dotnet add LibrarySeatReservation.Web package Microsoft.EntityFrameworkCore.Tools

# 3. 清理默认模板文件（删除 Bootstrap 5.x 本地包，改用 CDN）
# 保持 _Layout.cshtml 指向 CDN Bootstrap

# 4. 首次构建
dotnet build

# 5. 创建 Entity 模型（Models/Entity/）
# User.cs / Seat.cs / SeatArea.cs / Reservation.cs / Admin.cs

# 6. 创建 DbContext（Data/AppDbContext.cs）

# 7. 生成 Initial Migration
dotnet ef migrations add InitialCreate --project LibrarySeatReservation.Web

# 8. 首次运行
dotnet run --project LibrarySeatReservation.Web
```

## 四、Session / Cookie 使用原则

| 角色 | Session 键 | 用途 |
|------|-----------|------|
| 用户 | `UserId` | 标识当前体验用户 |
| 管理员 | `AdminId`, `AdminUsername` | 标识当前管理员 |

- 用户端：无需密码，点击体验账号直接写入 Session
- 管理端：表单输入用户名密码，验证后写入 Session
- 登出：清除对应 Session 键

## 七、默认补足项 / 当前假设

- **LocalDB 连接串**：`Server=(localdb)\MSSQLLocalDB;Database=LibrarySeatReservation;Trusted_Connection=true;TrustServerCertificate=true;` — 前序文档未指定连接串，按课堂默认
- **管理员密码**：种子数据使用明文存储（`passwordHash` 字段实际存明文），课堂项目不做哈希
- **端口**：使用 dotnet run 默认端口（Kestrel 自动分配），未固定端口号
- **HTTPS**：开发环境使用 dev cert，未在 appsettings 中固定配置
