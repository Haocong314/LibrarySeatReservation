# 图书馆座位预约系统 — 系统设计说明

## 1. 系统设计目标

1. 以 ASP.NET Core MVC (.NET 8) + Razor 为框架，搭建一个结构清晰、职责分明的三层架构（Controller → Service → DataAccess）。
2. 9 个核心页面与控制器一一对应，每条主链路从请求到数据库再回到视图，形成闭环。
3. 用户端和管理端共用同一套分层架构，通过不同的 Controller 文件夹和 Area（可选）隔离。
4. 代码目录结构对齐课堂项目规范，方便演示、答辩和后续迭代。

---

## 2. 技术栈说明

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 后端框架 | ASP.NET Core MVC (.NET 8) | 控制器处理请求，Razor 视图渲染页面 |
| 数据库 | SQL Server LocalDB | 轻量本地数据库，无需独立安装 SQL Server |
| ORM | Entity Framework Core 8 | Code First 迁移，自动建库建表 |
| 前端框架 | Bootstrap 5.3（CDN） | 响应式布局，用户端手机优先，管理端桌面优先 |
| 自定义样式 | `wwwroot/css/site.css` | 覆盖 Bootstrap 默认风格，统一校园服务感主题 |
| 认证 | Session / Cookie（管理端），体验账号下拉切换（用户端） | 轻量实现，不引入 ASP.NET Identity |
| 版本控制 | Git | 本地仓库 + GitHub/Gitee 远端（待补） |

## 4. 分层说明

### 4.1 Controller 层

**职责**：接收 HTTP 请求，调用 Service 获取数据，将 ViewModel 传给 View，返回 HTML。

**约束**：
- Controller 不做数据库查询（不直接调 DbContext）、不做业务判断。
- 每个 Action 方法对应一个页面或一个 POST 操作。
- 管理端 Controller 应在 Action 入口处检查 Session 登录状态，未登录跳转 P06。

### 4.2 Service 层

**职责**：承载全部业务逻辑。包括预约冲突检查、座位状态计算、统计汇总、登录验证。

**约束**：
- 每个 Service 通过构造函数注入 DbContext。
- 方法返回 ViewModel 或基础类型（bool、int、List 等），不返回 Entity 给 Controller。
- `ReservationService` 的 `CreateReservationAsync` 方法必须包含事务性冲突检查。

### 4.3 Data 层（DataAccess）

**职责**：仅包含 `AppDbContext` 和 EF Core 实体配置。不写 SQL，不包含业务逻辑。

**约束**：
- 所有数据库访问通过 DbSet + LINQ 完成。
- 实体关系配置在 `OnModelCreating` 中明确 Fluent API。

### 4.4 ViewModel 层

**职责**：作为 Controller 与 View 之间的数据契约。每个页面一个 ViewModel。

**约束**：
- ViewModel 不包含数据库逻辑。
- ViewModel 中的枚举值使用 `SelectListItem` 或简单字符串，避免 Razor 中硬编码。

### 4.5 Entity 层

**职责**：与数据库表结构一一映射。属性的名称和类型由数据库设计文档确定（见后续阶段 08）。

## 7. 主链路工程落点

### 7.1 用户预约主链路

```
GET  /Home/Index          → HomeController.Index()         → P01 首页
GET  /Seat/List           → SeatController.List()           → P02 座位列表
GET  /Seat/Detail/{id}    → SeatController.Detail(id)       → P03 座位详情
GET  /Reservation/Create?seatId={id} → ReservationController.Create(id) → P04 预约提交
POST /Reservation/Create  → ReservationController.Create(vm) → ReservationService.CreateReservationAsync()
                                  ├── 冲突检查（同一座位+日期+时段）
                                  ├── 通过 → 写入 Reservation 表
                                  └── 不通过 → 返回错误，停留在 P04
GET  /Reservation/MyReservations → ReservationController.MyReservations() → P05 我的预约
```

### 7.2 用户取消链路

```
POST /Reservation/Cancel/{id} → ReservationController.Cancel(id)
     → ReservationService.CancelAsync()
         ├── 检查预约是否属于当前用户
         ├── 检查预约状态是否为"已预约"
         ├── 更新状态为"已取消"
         └── 返回 MyReservations 视图
```

### 7.3 管理员管理链路

```
GET  /Admin/Login                    → P06 登录页
POST /Admin/Login                    → AdminService.Validate()
                                        ├── 成功 → 写入 Session，跳转 P07
                                        └── 失败 → 返回错误提示

GET  /AdminManage/Reservations?date=&status=&area= → AdminManageController.Reservations() → P07
POST /AdminManage/CancelReservation/{id}           → ReservationService.CancelAsync()

GET  /AdminManage/Seats              → P08 座位管理
POST /AdminManage/AddSeat / EditSeat / ToggleSeat  → SeatService

GET  /AdminManage/Stats              → AdminManageController.Stats() → AdminService.GetStats() → P09
```

## 8. 异常处理原则

| 异常类型 | 处理方式 |
|----------|----------|
| 预约冲突 | Controller 调用 Service 返回 false，View 显示"该时段已被预约，请选择其他时段" |
| 座位不存在 | 返回 P02 座位列表，显示"座位已不存在或已停用" |
| 取消已取消的预约 | Service 检查状态，"已取消"则返回 false，View 提示"该预约已取消，无需重复操作" |
| 取消他人预约 | Service 检查预约所属用户，不匹配则拒绝，View 提示"无权操作" |
| 管理员未登录访问后台 | 检查 Session["AdminId"] 是否为 null，跳转到 P06 |
| 数据库连接失败 | 全局异常处理中间件捕获，返回友好错误页"系统繁忙，请稍后重试" |
| 模型绑定失败 | ASP.NET Core 内置 ModelState，View 中通过 `asp-validation-summary` 展示错误 |

## 9. Session / Cookie 使用原则

### 9.1 用户端：体验账号

- 不存 Session，不写 Cookie。
- 页面顶部下拉框切换账号，将 `currentUserId` 存在 ViewBag 或 TempData 中。
- 每次请求时 Controller 从 QueryString 或隐藏字段读取当前用户 ID。
- 简化方案：所有用户端页面共享一个固定的体验账号 ID 列表，切换即为切换 ID。

### 9.2 管理端：Session 登录

- 管理员登录成功后写入 `HttpContext.Session.SetString("AdminId", admin.Id.ToString())`。
- `AdminManageController` 每个 Action 入口检查 Session，不存在则 `RedirectToAction("Login", "Admin")`。
- 退出时清除 Session：`HttpContext.Session.Clear()`。
- 不使用 Cookie 持久化登录（浏览器关闭即退出）。

### 9.3 配置要点

- `Program.cs` 中注册：`builder.Services.AddSession();` + `app.UseSession();`
- Session 超时使用默认 20 分钟即可。
