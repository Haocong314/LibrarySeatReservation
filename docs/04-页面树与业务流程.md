## 2. 用户端页面树

```
[用户首页 P01]
 ├── 点击"找座位" ──→ [座位列表页 P02]
 │     └── 点击某个座位卡片 ──→ [座位详情页 P03]
 │           └── 点击"去预约" ──→ [预约提交页 P04]
 │                 └── 提交成功 ──→ [我的预约页 P05]
 │
 ├── 点击"我的预约" ──→ [我的预约页 P05]
 │     └── 点击"取消" → 确认 ──→ 刷新 [我的预约页 P05]
 │
 └── 顶部"切换账号"下拉框 ──→ 刷新当前页面（切换用户身份）
```

| 起点 | 操作 | 终点 | 说明 |
|------|------|------|------|

| P01 用户首页 | 点击"我的预约" | P05 我的预约页 | 快捷入口 |
| P02 座位列表页 | 点击座位卡片 | P03 座位详情页 | 带座位 ID 参数 |
| P02 座位列表页 | 点击返回 | P01 用户首页 | 顶部返回按钮 |
| P03 座位详情页 | 点击"去预约" | P04 预约提交页 | 带座位 ID 参数 |
| P03 座位详情页 | 点击返回 | P02 座位列表页 | 顶部返回按钮 |
| P04 预约提交页 | 提交成功 | P05 我的预约页 | 自动跳转 |
| P04 预约提交页 | 提交失败（冲突） | P04 预约提交页（本页） | 显示错误提示，不跳转 |

[管理员登录页 P06]

        │
        └── 左侧导航"统计" ──→ [统计页 P09]

未登录状态下访问 P07/P08/P09 ──→ 重定向到 [管理员登录页 P06]

## 4. 用户主链路

### 链路名称：学生预约座位（含取消）

```
步骤 1：P01 用户首页
  → 用户选择体验账号（下拉框切换身份）
  → 点击"找座位"按钮

步骤 2：P02 座位列表页
  → 看到座位按区域分组展示
  → 每条座位卡片显示：座位号、区域名称、状态标签（可预约/已满/停用）
  → 点击一个"可预约"状态的座位卡片

步骤 3：P03 座位详情页
  → 展示座位详细信息：座位号、区域、楼层、设施（电源/台灯）
  → 展示该座位今天各时段预约状态
  → 点击"去预约"按钮

步骤 4：P04 预约提交页
  → 顶部显示已选座位信息（确认没选错）
  → 选择日期（日期选择器或下拉框）
  → 选择时段（下拉框，展示可用时段）
  → 点击"提交预约"
  → 后端执行冲突检查：
      - 无冲突 → 写入 Reservation 表 → 跳转 P05
      - 有冲突 → 本页显示错误提示"该时段已被预约"

步骤 5：P05 我的预约页
  → 看到刚刚提交的预约记录（状态为"已预约"，绿色标签）
  → 列表按时间倒序排列
  → 如需取消：点击某条预约的"取消"按钮
      → 弹出确认对话框
      → 确认后该条状态变为"已取消"（灰色标签）
      → 该座位该时段释放，其他用户可预约
```

**闭环验证**：P01 → P02 → P03 → P04 → P05，五步形成完整预约闭环。取消操作在 P05 完成闭环。

## 5. 管理主链路

### 链路名称：管理员登录 → 预约管理 → 座位管理 → 统计

```
步骤 1：P06 管理员登录页
  → 输入管理员账号和密码
  → 点击"登录"
  → 验证通过 → 写入 Session → 跳转 P07
  → 验证失败 → 本页提示"账号或密码错误"

步骤 2：P07 预约管理页（默认后台首页）
  → 顶部显示当前登录管理员信息 + "退出登录"
  → 筛选区：日期选择器、状态下拉框、区域下拉框
  → 表格区：用户、座位号、区域、日期、时段、状态、操作
  → 筛选条件变更 → 表格数据即时刷新
  → 可对异常预约执行取消操作

步骤 3：P08 座位管理页（左侧导航进入）
  → 表格展示全部座位：座位号、区域、楼层、设施、状态、操作
  → "新增座位"按钮 → 弹出表单 → 填写 → 提交 → 列表刷新
  → "编辑"按钮 → 弹出表单 → 修改 → 提交 → 列表刷新
  → "停用/启用"按钮 → 确认 → 状态切换 → 列表刷新

步骤 4：P09 统计页（左侧导航进入）
  → 数据卡片：今日预约总数、活跃用户数
  → 区域分布表格：各区域今日预约数
  → 数据和 P07 按今天日期筛选的结果一致
```

**闭环验证**：P06 → P07 → P08 → P09，管理功能全覆盖。

## 6. 状态流转

### 6.1 预约状态（Reservation.Status）

```
                用户提交预约
   [无] ─────────────────────→ [已预约 Active]
                                    │
                    ┌───────────────┼───────────────┐
                    │               │               │
               用户取消         管理员取消        （未来：到期自动完成）
                    │               │
                    ↓               ↓
              [已取消 Cancelled]  [已取消 Cancelled]
```

- **已预约 Active**：预约生效中，该座位该日期该时段被占用
- **已取消 Cancelled**：预约已取消，座位释放，不可恢复
- 已取消的预约在列表中保留记录，以灰色标签展示

### 6.2 座位状态（Seat.Status）

```
   [可用 Available] ←── 管理员"启用" ──→ [停用 Disabled]
          │
          └── 管理员"停用" ──→ [停用 Disabled]
```

- **可用 Available**：用户端可见，可以预约
- **停用 Disabled**：用户端显示为"停用"标签，不可预约；已有预约不受影响

## 7. 页面与流程关系说明

| 流程环节 | 涉及页面 | 数据操作 | 状态变化 |
|----------|----------|----------|----------|
| 浏览座位 | P01 → P02 | 读：SeatArea, Seat | 无 |
| 查看详情 | P02 → P03 | 读：Seat, TimeSlot, Reservation（查已占时段） | 无 |
| 提交预约 | P03 → P04 | 写：Reservation（INSERT） | Reservation 状态 = Active |
| 确认结果 | P04 → P05 | 读：Reservation（当前用户） | 无 |
| 取消预约 | P05 | 写：Reservation（UPDATE Status） | Reservation: Active → Cancelled |
| 管理登录 | P06 → P07 | 读：User（验证） + 写：Session | Session 创建 |
| 预约管理 | P07 | 读：Reservation（筛选查询） | 无 / 管理员取消时变 Cancelled |
| 座位管理 | P08 | 写：Seat（CRUD） | Seat.Status: Available ↔ Disabled |
| 查看统计 | P09 | 读：Reservation（聚合查询） | 无 |
